Take stock of the kitchen
Dish 1 🍳
🍳 Every chef starts service by walking the kitchen. Your first mission: ask your sous-chef to do exactly that.
In VS Code, open the Claude Code panel (click the ✦ sparkle icon). Type this and press Enter:
TYPE →what files and folders are in my current directory?
What to expect: Claude Code lists everything it sees. That's your sous-chef doing the first walk of the kitchen.
2
Prep your first ingredient
Dish 2 🥘
🍳 Time to make something. Your sous-chef will create a real file — and will ask your permission before touching anything.
Type this in Claude Code:
TYPE →create a file called my-notes.md and write a short summary of what Claude Code is
What to expect: Claude Code describes what it's about to do and asks for confirmation. Say yes. To verify it worked — look at the Explorer panel (📁 icon in the left sidebar). You should see my-notes.md appear there.
3
Taste and adjust
Dish 3 🧑‍🍳
🍳 A great chef doesn't just cook — they taste, critique, and improve. Two steps: get feedback, then act on one suggestion.
Step 1 — ask for a review:
TYPE →read my-notes.md and tell me 2 things you'd improve about it
Then
Step 2 — act on the first suggestion:
TYPE →now make the first improvement to the file
What to expect: First it critiques its own writing. Then it rewrites part of the file. Check the Explorer — the file is updated.
4
Plan the week's menu
Dish 4 📋
🍳 Give your sous-chef something bigger. Ask it to look at your whole kitchen and suggest what to build — then do one of them.
Step 1 — get suggestions first:
TYPE →look at my current folder and suggest 3 things I could build or organize this week. Do NOT start building yet — just give me the suggestions first.
Then
Step 2 — pick one and start:
TYPE →let's do the first one — go ahead and build it
What to expect: Suggestions based on what's actually in your folder. Then your sous-chef starts on whichever you choose.
5
File the recipe (save to GitHub)
Dish 5 🗂️
🍳 After a good service, you file the recipes so any chef can recreate them. We'll create a dedicated project folder first — never push from Claude's config folder.
Step 1 — set up a clean station:
TYPE →create a new folder called my-first-claude-project in my Documents folder and initialize it as a git repository
Then
Step 2 — add a README and push to a public repo:
TYPE →create a README.md in this folder that says: 'This is my first Claude Code project.' Then commit everything and push to a new public GitHub repository called my-first-claude-project
⚠️ Why a new folder? Claude Code's config folder contains your private credentials. Never push that to GitHub. Always work in a dedicated project folder.
💡 Why public? Public repos get GitHub Pages for free — no premium account needed.
What to expect: Claude Code will ask permission at each step. At the end, you get a GitHub link you can share.
6
Daily closing routine
Dish 6 🔁
🍳 Every great kitchen has a closing routine — update the logbook, clean the station, prep for tomorrow.
At the end of every session, type this:
TYPE →summarize what we cooked today, save it to my-notes.md in my-first-claude-project, and commit everything to GitHub with a note about today's session
What to expect: Your sous-chef writes a session summary, saves it, and commits to GitHub in one go.

Cook the first dish

Taste and adjust

Then
Keep going until it feels right:

Design the game
Dish 1 🎮
🎮 Every great game starts with a brief. Before a single line of code, you're the creative director.
Tell Claude Code what kind of game you want. Add 'do NOT start building yet':
TYPE →I want to build a browser game called Chef's Chaos. Ingredients fall from the top of the screen and I move a pot left and right to catch the right ones. Wrong ingredients cost a life. Speed increases over time. Ask me anything you need to know. Do NOT start building yet — just ask your questions first.
Then
Answer its questions, then ask for a game plan:
TYPE →based on what I told you, write a short game design doc — rules, ingredients list, scoring, and visual style
What to expect: Claude Code asks clarifying questions, then produces a game design document that becomes your blueprint.
2
Cook the engine
Dish 2 ⚙️
🍳 The design is ready. Now your sous-chef builds the engine — falling ingredients, a moving pot, collision detection.
Ask Claude Code to build the core game:
TYPE →build the core Chef's Chaos game based on our design doc. Create a single index.html file with all the HTML, CSS and JavaScript inside it. The pot moves with arrow keys. Make it playable but rough — we'll polish next.
Then
Once done, open it:
TYPE →open the game in my browser so I can test it
What to expect: A working game opens in your browser. It will look rough. The only question: does it move? Can you catch things? Does the score go up?
3
Add the drama
Dish 3 🎭
🎭 A working game is not yet a fun game. Drama is what makes people keep playing.
Test the game, then ask Claude Code to add tension:
TYPE →add the following to the game: 3 lives shown as heart emojis, a game over screen with my final score and a play again button, speed that increases every 10 points, and one special golden ingredient worth 5 points
Then
If something doesn't feel right:
TYPE →the game feels [too fast / too slow / too easy / too hard] — adjust it so it feels satisfying to play
What to expect: Each addition makes the game feel more real. Describe what feels off in plain language — it gets fixed.
4
Plate it beautifully
Dish 4 🎨
🎨 The game works. Now make it look like something you'd actually want to play.
Describe the visual style you want:
TYPE →make the game look polished and fun. Add: a start screen with the game title and a big play button, a kitchen background, emoji ingredients that wiggle as they fall, and a sound effect when you catch something correctly. Keep it fast to load.
Then
Keep adjusting until it feels right:
TYPE →change the [colour / font / background / animation speed] to [what you want]
What to expect: The game starts looking like a real product. These are the details that make someone say 'did you actually build this?'

Open the arcade

Then
Step 2 — enable GitHub Pages:
TYPE →tell me how to enable GitHub Pages for my chefs-chaos repository so the game goes live
What to expect: Claude Code pushes the files, walks you through GitHub Pages. Within 1-2 minutes your game is live. Open it on your phone.
